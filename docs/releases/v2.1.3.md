# Web Search Pro 2.1.3 Release Pack

This file contains release materials for both GitHub and ClawHub.

## Release Summary

`Web Search Pro 2.1.3` is the ClawHub metadata-alignment follow-up to `2.1.2`.

It does not change routing, providers, or retrieval behavior. Instead, it makes the published
package declare three things more explicitly:

- this is a code-backed Node skill pack
- it may read `config.json` as an optional runtime config file
- it may write local state under `.cache/web-search-pro`

Product / docs line remains `2.1`.  
Release tag advances to `2.1.3`.  
Stable JSON schema remains `1.0`.

## GitHub Release Notes

### Title

`v2.1.3 - Align ClawHub install and config metadata with the shipped runtime`

### Body

`Web Search Pro 2.1.3` tightens the registry-facing metadata for the ClawHub package.

This patch release focuses on metadata honesty, not new retrieval features:

- explicit install-model wording for the bundled Node skill pack
- explicit config-path disclosure for `config.json`
- explicit state-directory disclosure for `.cache/web-search-pro`

### Highlights

- Added an `Install Model` section to `SKILL.md` and generated ClawHub docs
- Added `clawdbot.requires.config = ["config.json"]`
- Added `clawdbot.config.stateDirs = [".cache/web-search-pro"]`
- Added a direct Node-backed `clawdbot.install` declaration
- Updated ClawHub compliance and packaging docs
- Added TDD coverage for the expanded metadata contract

### Why This Release Matters

Before `2.1.3`, the ClawHub package already declared the Node runtime and no-key baseline, but it
still under-described two real parts of its runtime shape:

- the optional `config.json` read path
- the local cache / health state directory

`2.1.3` makes those local state and install-model details explicit so registry review has a more
honest description of the shipped package.

### Compatibility

- Product / docs line remains `2.1`
- Release version is `2.1.3`
- Stable machine-readable JSON schema remains `1.0`
- Search / extract / crawl / map / research behavior remains unchanged

## ClawHub Release Description

`Web Search Pro 2.1.3` aligns the ClawHub package metadata with the runtime it actually ships.

This patch release adds:

- an explicit Node-backed install model
- explicit `config.json` disclosure
- explicit `.cache/web-search-pro` state-directory disclosure
- matching docs and tests for the package contract

The core package remains the same:

- search
- extract
- crawl
- map
- research
- bootstrap / doctor / capabilities / review
- cache / health

No API key is required for the baseline. Optional keys or endpoints still widen provider coverage.
