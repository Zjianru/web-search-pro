# Web Search Pro 2.1.4 Release Pack

This file contains release materials for both GitHub and ClawHub.

## Release Summary

`Web Search Pro 2.1.4` is the documentation-clarity follow-up to `2.1.3`.

It does not change search quality, provider coverage, or retrieval behavior. Instead, it makes
the project easier to understand on first read by separating:

- English human/developer onboarding
- Chinese human/developer onboarding
- agent-facing runtime contract docs

Product / docs line remains `2.1`.  
Release tag advances to `2.1.4`.  
Stable JSON schema remains `1.0`.

## GitHub Release Notes

### Title

`v2.1.4 - Split docs by audience and language`

### Body

`Web Search Pro 2.1.4` is a documentation-focused patch release.

This release improves first-run understanding without changing the runtime:

- `README.md` is now an English-only human/developer landing page
- `README_zh.md` is now a standalone Chinese landing page
- `SKILL.md` is framed more clearly as an agent-facing runtime contract
- core docs now state their intended audience more explicitly

### Highlights

- Added a dedicated Chinese README with the same product story, not a mixed bilingual page
- Added explicit language switcher links between English and Chinese README surfaces
- Rewrote the top-level onboarding flow so developers can understand:
  - what the skill is
  - when to choose it
  - when not to choose it
  - how to get the first successful run
- Audited supporting docs and ClawHub package doc generation for audience clarity
- Added test coverage for the split README contract

### Why This Release Matters

Before `2.1.4`, the repository docs had the right facts but mixed too many concerns together:

- human onboarding
- agent contract explanation
- bilingual copy
- internal implementation context

That made the first screen harder to scan, especially for developers evaluating whether the skill
fit their workflow.

`2.1.4` fixes that by separating surfaces instead of simplifying the architecture dishonestly.

### Compatibility

- Product / docs line remains `2.1`
- Release version is `2.1.4`
- Stable machine-readable JSON schema remains `1.0`
- Search / extract / crawl / map / research behavior remains unchanged
