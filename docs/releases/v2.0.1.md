# Web Search Pro 2.0.1 Release Pack

This file contains release materials for both GitHub and ClawHub.

## Release Summary

`Web Search Pro 2.0.1` is the compliance and packaging follow-up to `2.0.0`.

The full repository keeps the complete `2.0` source tree, but ClawHub now publishes a generated
core profile so registry-facing metadata and scan scope stay closer to the installed artifact.

Product version remains `2.0.x`.  
Stable JSON schema remains `1.0`.

## GitHub Release Notes

### Title

`v2.0.1 - Add a ClawHub core package profile`

### Body

`Web Search Pro 2.0.1` is the compliance and packaging follow-up to `2.0.0`.

The repository still contains the full `2.0` retrieval stack, including local render, eval, and
the full test / research toolchain. The change in `2.0.1` is that ClawHub publishing now uses a
generated core profile instead of the raw repository root.

### Highlights

- Added `scripts/build-clawhub-package.mjs`
- Added `scripts/lib/clawhub-package.mjs`
- Added `docs/clawhub-package.md`
- Added TDD coverage for the generated package contract
- Added optional provider env disclosure to the generated ClawHub package metadata
- Removed `tests`, `eval`, and the explicit render lane from the generated ClawHub package

### What Did Not Change

- GitHub source remains the full `2.0` feature set
- local OpenClaw usage remains the full `2.0` feature set
- stable JSON schema remains `1.0`
- core commands and research-pack behavior remain intact

### Why This Matters

The live ClawHub moderation signal was not caused by a single false positive. It came from a
mismatch between:

- the full repository surface
- the package artifact the registry scans
- the metadata the registry can reason about

`2.0.1` fixes that by publishing a narrower, more honest ClawHub package without pretending the
root repository is smaller than it really is.

## ClawHub Release Description

`Web Search Pro 2.0.1` publishes a registry-friendly core profile of the full `2.0` retrieval
stack.

The ClawHub package keeps:

- search
- extract
- crawl
- map
- research
- doctor / capabilities / review
- cache / health

It keeps the no-key baseline and optional provider enhancements, while removing registry-noisy
developer surfaces from the published artifact.

No API key is required for the baseline. Optional provider keys widen retrieval quality through
Tavily, Exa, Serper, and SerpAPI.

## ClawHub Maintainer Compliance Note

Maintainer note: `web-search-pro 2.0.1` now publishes a generated ClawHub core profile instead of
the raw repository root.

Why:

- the full repository contains additional local-only developer surfaces
- ClawHub moderation reasons about the published folder, not just runtime intent
- a narrower publish artifact better matches what the registry can honestly review

The published ClawHub package:

- declares only the real hard runtime requirement: `node`
- discloses optional provider env vars directly in package metadata
- keeps the no-key baseline
- keeps review-visible runtime diagnostics through `capabilities.mjs`, `doctor.mjs`, and `review.mjs`
- removes test, eval, and explicit browser-render surfaces from the registry package

Full source code remains public here:

https://github.com/Zjianru/web-search-pro
