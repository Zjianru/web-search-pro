# Web Search Pro 2.0.0 Release Pack

This file contains release materials for both GitHub and ClawHub.

## Release Summary

`Web Search Pro 2.0.0` is the major evolution of the original `1.x` line.

The old line centered on multi-provider search and extraction.  
`2.0.0` keeps that foundation, but reorganizes the product into a retrieval system for agents and
upstream models:

- retrieval primitives: search, extract, render, crawl, map
- execution control: routing, federation, cache, provider health
- research substrate: `plan + evidence pack`
- operator and review surface: doctor, capabilities, review, cache, health, eval

Product version is now `2.0.0`.  
Stable JSON schema remains `1.0`.

## GitHub Release Notes

### Title

`v2.0.0 - Web Search Pro becomes a retrieval stack`

### Body

`Web Search Pro 2.0.0` is the major evolution of the original `1.x` line.

This project started as a multi-provider search supplement. In `2.0.0`, it becomes a full
retrieval stack for agents and upstream models.

### Highlights

- Added a real no-key baseline with `ddg` search and `fetch`-based extract / crawl / map
- Added `crawl.mjs` and `map.mjs` for site-level discovery and retrieval
- Added an optional local browser render lane via `render.mjs`
- Added controlled federated retrieval with merge, dedupe, rerank, and compact merge telemetry
- Added runtime diagnostics and review surfaces: `doctor`, `capabilities`, `review`, `cache`, and `health`
- Added `research.mjs` for structured `plan + evidence pack` generation
- Added bundled `eval` coverage for both retrieval and research behavior

### From 1.x To 2.0

`1.x` focused on:

- multi-engine search
- provider-specific flags
- extraction

`2.0.0` expands that into:

- search
- extract
- render
- crawl
- map
- federation
- cache / health / review
- research evidence packs

### Compatibility

- Product version is now `2.0.0`
- Stable machine-readable JSON schema remains `1.0`
- Existing `search` and `extract` consumers remain valid
- New fields such as `federated`, `render`, and research-pack structures are additive

### Compliance And Safety

- metadata still declares only the real hard requirement: `node`
- provider credentials remain optional
- provider disclosure moves through `capabilities`, `doctor`, and `review`
- safe fetch and browser render boundaries are explicit
- provider health now distinguishes `degraded` from `cooldown`

### Notes

- `ddg` remains a best-effort baseline and may be affected by external anti-bot challenge pages
- `research.mjs` is a structured evidence-pack generator, not a final narrative report writer

## ClawHub Release Description

`Web Search Pro 2.0.0` is the major evolution of the original `1.x` line.

It is no longer just a multi-provider search script. The current product is a retrieval stack for
agents and upstream models:

- multi-provider search with explainable routing
- no-key baseline retrieval
- single-page extract plus optional browser render
- BFS crawl and site map
- controlled federated retrieval with merge / dedupe / rerank
- structured `research` evidence packs
- diagnostics and review output for runtime validation

No API key is required for the baseline. Optional provider keys unlock enhanced features through
Tavily, Exa, Serper, and SerpAPI.

## ClawHub Maintainer Compliance Note

Maintainer note: `web-search-pro 2.0.0` keeps the current compliance posture intentionally.

This skill supports a no-key baseline plus multiple optional providers. Different commands and
flags use different provider paths, so the runtime contract is a capability matrix, not a single
"all env vars required" gate.

For that reason, metadata only declares the real hard runtime requirement: `node`.

Provider use is disclosed through runtime review surfaces instead of hidden:

- `capabilities.mjs`
- `doctor.mjs`
- `review.mjs`

The code only uses documented provider credentials and documented provider APIs. Safe fetch and
browser render boundaries are explicit, reviewable, and exposed through command output.
