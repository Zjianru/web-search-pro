# Web Search Pro 2.1.1 Release Pack

This file contains release materials for both GitHub and ClawHub.

## Release Summary

`Web Search Pro 2.1.1` is the first patch release after `2.1.0`.

It does not add a new architectural surface. Instead, it strengthens two things that matter a lot
for real-world agent adoption:

- lower first-use friction through a clearer quick-start path
- visible federated-search gains through compact output metrics

Product / docs line remains `2.1`.  
Release tag advances to `2.1.1`.  
Stable JSON schema remains `1.0`.

## GitHub Release Notes

### Title

`v2.1.1 - Improve onboarding and federated-search visibility`

### Body

`Web Search Pro 2.1.1` makes the existing retrieval stack easier to adopt and easier to trust on
the first run.

This patch release focuses on two practical improvements:

- clearer quick-start documentation for the no-key baseline and one-provider upgrade path
- concrete federated-search value metrics that show what multi-provider search actually added

### Highlights

- Added a real quick-start flow to `README.md`:
  - `Option A: No-key baseline`
  - `Option B: Add one premium provider`
  - `First successful searches`
  - `Then try docs, news, and research`
- Added the same quick-start guidance to `SKILL.md`
- Synced generated ClawHub package docs with the same onboarding structure
- Added compact federation gain metrics:
  - `federated.value.additionalProvidersUsed`
  - `federated.value.resultsRecoveredByFanout`
  - `federated.value.resultsCorroboratedByFanout`
  - `federated.value.duplicateSavings`
- Updated markdown search output so federated runs now explain the gain directly

### Why This Release Matters

Before `2.1.1`, the runtime already had:

- no-key baseline search and retrieval
- explainable routing
- federated fanout
- broader provider coverage

But two things were still weaker than they should be:

- first-time users had to infer the minimum successful path from command lists
- federated search exposed raw merge facts, but not a compact explanation of why it beat
  primary-only search

`2.1.1` fixes both without changing the broader architecture.

### Compatibility

- Product / docs line remains `2.1`
- Release version is `2.1.1`
- Stable machine-readable JSON schema remains `1.0`
- Existing consumers remain valid
- New `federated.value.*` fields are additive

## ClawHub Release Description

`Web Search Pro 2.1.1` makes the current retrieval stack easier to adopt and easier to interpret.

This patch release adds:

- a more explicit quick start for the no-key baseline
- a simpler path for adding one premium provider first
- visible federated-search gain metrics in search output
- synchronized onboarding copy across GitHub, SKILL, and generated ClawHub docs

The generated ClawHub package still keeps the same core profile:

- search
- extract
- crawl
- map
- research
- bootstrap / doctor / capabilities / review
- cache / health

No API key is required for the baseline. Optional keys or endpoints still unlock premium retrieval
through Tavily, Exa, Querit, Serper, Brave, SerpAPI, You.com, SearXNG, and Perplexity / Sonar.

## ClawHub Maintainer Compliance Note

Maintainer note: `web-search-pro 2.1.1` does not change the current publish boundary.

Why:

- GitHub remains the canonical full source tree
- ClawHub continues to publish the generated core profile
- the only hard runtime requirement is still `node`
- optional provider envs remain optional

What changed in the generated package is not the boundary, but the clarity:

- clearer first-run guidance
- a more visible no-key baseline path
- explicit federation gain terminology for installed agents
