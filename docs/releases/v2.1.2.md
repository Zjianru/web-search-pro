# Web Search Pro 2.1.2 Release Pack

This file contains release materials for both GitHub and ClawHub.

## Release Summary

`Web Search Pro 2.1.2` is the ClawHub trust-hardening follow-up to `2.1.1`.

It does not add new search features. Instead, it tightens the published runtime story so registry
review sees the same thing users actually install and run:

- a code-backed Node skill package
- a real no-key baseline
- no Python helper transport in the shipped baseline path

Product / docs line remains `2.1`.  
Release tag advances to `2.1.2`.  
Stable JSON schema remains `1.0`.

## GitHub Release Notes

### Title

`v2.1.2 - Harden ClawHub packaging and baseline runtime honesty`

### Body

`Web Search Pro 2.1.2` improves the trust and clarity of the published package without changing
the search-router or retrieval-stack surface.

This patch release focuses on three things:

- making the ClawHub package metadata align with the actual shipped runtime
- describing the package explicitly as a code-backed Node runtime, not an instruction-only bundle
- removing the Python helper transport from the no-key DuckDuckGo baseline path

### Highlights

- Added `metadata.clawdbot` alongside compatibility `metadata.openclaw`
- Added `cliHelp` for the ClawHub-facing package metadata
- Updated generated ClawHub README / SKILL copy to explain the shipped runtime shape more clearly
- Switched `ddg` baseline POST requests to `curl`
- Kept built-in `fetch` as the baseline fallback path
- Removed the Python helper transport implementation from `http-client.mjs`
- Updated ClawHub compliance and packaging docs to match the current publish surface

### Why This Release Matters

Before `2.1.2`, the runtime behavior was already functional, but the published package could still
look stranger than it really was to static review:

- metadata was clearer for OpenClaw than for ClawHub
- the package did not say loudly enough that it is a real Node runtime package
- the no-key baseline still contained a Python helper transport that raised unnecessary suspicion

`2.1.2` fixes those trust and packaging mismatches without weakening baseline retrieval.

### Compatibility

- Product / docs line remains `2.1`
- Release version is `2.1.2`
- Stable machine-readable JSON schema remains `1.0`
- Existing search / extract / crawl / map / research flows remain valid
- The no-key baseline still works through `ddg` plus safe fetch-based retrieval

## ClawHub Release Description

`Web Search Pro 2.1.2` tightens the published package story for ClawHub.

This patch release:

- adds `metadata.clawdbot` for clearer registry-facing metadata
- describes the package explicitly as a code-backed Node runtime
- removes the Python helper transport from the baseline `ddg` path
- keeps the same no-key baseline and core retrieval profile

The ClawHub package still ships:

- search
- extract
- crawl
- map
- research
- bootstrap / doctor / capabilities / review
- cache / health

No API key is required for the baseline. Optional keys or endpoints still unlock premium retrieval
through Tavily, Exa, Querit, Serper, Brave, SerpAPI, You.com, SearXNG, and Perplexity / Sonar.
